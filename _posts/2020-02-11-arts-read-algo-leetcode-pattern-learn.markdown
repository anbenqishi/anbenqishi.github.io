##概述

编码面试（技术面，涉及到算法的）对于大部分开发者来说容易引发焦虑，因为要覆盖的范围太广了。很多看起来跟日常工作都是无关的，无疑更增添了压力。

当前大部分码农采取的方式是大量地刷题，如Leetcode。不过，还是有求职焦虑症：我解决了足够多的问题了吗？我可以做更多吗？

这里总结了14个模式，用来解决面试时的算法问题，避免陷入到Leetcode的题海中去。
